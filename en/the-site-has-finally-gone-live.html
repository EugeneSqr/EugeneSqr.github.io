The Site Has Finally Gone Live!

2014-07-28

At last

I'm excited to announce my site's first release! Although it isn't feature rich I consider this event as a very important milestone in my development career.

It's all began in 2012 when I first published my thoughts into Internet using blogspot blogging platform. I was writing mainly about .net stuff and only in English.

A year ago I came to conclusion that I didn't want to be a pure .net developer. Instead I wanted to expand the boundaries and explore things outside of Microsoft's ecosystem. I installed Linux and started playing around Node, Mongo, Angular, JavaScript, C and vim.

Two month before this day I decided it's time to examine my new skills. By that time I had something around 20 posts in my blog and I wasn't really happy how blogger was serving them. So I started working on this site during my spare time.

What are the key features?

This site is fast.

  1. I keep the number of dependencies to an absolute minimum so I can have full understanding of what's going on and can fix any problems without being locked in third party tool restrictions.
  2. I don't implement anything I don't really need. If something is no longer needed, it gets fully removed.
  3. I treat the content as a main value of this site; therefore I don't use any "WYSIWYG" editors to produce result html (which in many cases is far from ideal). Instead I balance productivity with markup clarity by using markdown.

The site is bilingual

I maintain two versions (English and Russian) of content plus static labels and headers. It basically means two things:

  1. I don't wan't to write something trivial or duplicate someone's articles; because in this case I have to translate it which is tedious. It's a good motivation to focus on my personal experience.
  2. I have to rethink a post each time I translate it; which also improves quality (especially spelling :)). Let's not forget that English is my second language so it's a good opportunity to improve my writing skills. If you found any problems in text (grammar, style etc.) please don't hesitate to tell me.

Responsive design

I use bootstrap 3.0 + less to manage my css. This results in terse code that works well on majority of devices.

All tools and components are free.

I use:

  • Linux Mint Debian Edition as development OS;
  • vim as a text editor;
  • NodeJS, ExpressJS - backend; AngularJS - frontend; Mongo - data storage. JavaScript everywhere!
  • misc: git, selenium, markdown, jasmine, bootstrap, less, bower and others.

The absence of proprietary software makes the project really personal.

Future development

Here is the list of must have features I plan to add next release:

  1. Google analytics;
  2. Post list paging;
  3. Spell checking strategy;
  4. Client side dates localization in angular;
  5. Improve storage structure to reduce content duplication across languages;
  6. Migrate more articles from blogger.com

That's it hope you enjoy it.
